Key Features

⚡ Sub-millisecond Retrieval

HNSW indexes deliver ~0.5ms vector search latency for millions of embeddings.

🎯 Semantic Caching

Cache LLM responses by semantic similarity, not exact match. 90%+ hit rates.

📊 Metadata Filtering

Combine vector search with tag, date, and category filters in a single query.

🔄 TTL Management

Automatic cache expiration with configurable TTLs per document type.

Quick Start

Create a vector index and perform semantic search with Valkey:

# Create a vector search index FT.CREATE idx:docs ON HASH PREFIX 1 "doc:" SCHEMA content TEXT section TEXT embedding VECTOR HNSW 6 TYPE FLOAT32 DIM 1536 DISTANCE_METRIC COSINE # Store a document chunk with embedding HSET doc:chunk_001 content "Valkey supports HNSW vector indexes..." section "Vector Search" embedding [binary_embedding_data] # Semantic search with KNN FT.SEARCH idx:docs "*=>[KNN 5 @embedding $vec AS score]" PARAMS 2 vec [query_embedding] SORTBY score RETURN 3 content section score DIALECT 2
